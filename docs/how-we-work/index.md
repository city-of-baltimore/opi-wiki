# How We Work

OPI moves work from a problem to a fix. This section explains the public
operating model and shows how the office is organized.

For the full model, read **[How Work Moves Through OPI](how-work-moves-through-opi.md)**. It covers what each service owns, how the handoffs work, and when to use a Stat, a data product, an Innovation Lab engagement, a Tiger Team, a Cross-Agency Delivery activation, or a decision memo. For each service in its own right, see **[Services](../what-we-do/services/index.md)**.

Internal onboarding, administrative guidance, how-to instructions, and
personnel records are maintained in the City's SharePoint environment and are
not published on this site.
